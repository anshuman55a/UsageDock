# Security Policy

## Supported Versions

UsageDock is maintained on the latest release line. Older versions may not receive security fixes.

## Reporting a Vulnerability

Please do not open public GitHub issues for security reports.

Report security issues privately to the maintainer through GitHub security advisories if enabled, or by contacting the repository owner directly. Include:

- affected version
- platform
- steps to reproduce
- impact
- any proof-of-concept or logs that help confirm the issue

You can expect:

- acknowledgement after the report is seen
- validation before a fix is planned
- a coordinated fix and release when the issue is confirmed

## Scope

Relevant issues include:

- credential leakage
- insecure token storage or transport
- local privilege or trust-boundary problems
- updater trust/signing issues
- provider integrations that bypass expected security controls

General product bugs should go through the normal issue tracker.
