## Auto Update Setup

`UsageDock` uses the Tauri updater for app updates. This is separate from Windows code-signing and SmartScreen.

### What the repo already does

- Local builds keep working with no updater secrets.
- Tagged GitHub releases generate `latest.json` and signed updater artifacts only when updater secrets are present.
- Release binaries compile the updater endpoint and public key into the app at build time.

### Required GitHub secrets

Add these repository secrets before expecting auto-update to work from CI:

- `TAURI_SIGNING_PRIVATE_KEY`
  The full private updater key generated by the Tauri signer.
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`
  The password for that private key. Leave it empty only if you intentionally created an unencrypted key.
- `TAURI_UPDATER_PUBLIC_KEY`
  The matching public key that the app embeds for signature verification.

The workflow already uses this endpoint for shipped builds:

- `https://github.com/anshuman55a/UsageDock/releases/latest/download/latest.json`

### Generate updater keys

Run this once on a trusted machine:

```powershell
npm run tauri signer generate
```

Store the generated private key in `TAURI_SIGNING_PRIVATE_KEY` and the public key in `TAURI_UPDATER_PUBLIC_KEY`.

### Release behavior

- If the updater secrets exist:
  - CI enables `createUpdaterArtifacts`
  - signs updater bundles
  - uploads `latest.json`
  - compiles updater support into the release app
- If the updater secrets are missing:
  - CI falls back to the current plain installer release flow
  - the in-app updater stays hidden

### Important notes

- This updater signing is not the same as Authenticode signing for Windows installers.
- Auto-update can work without buying Windows code-signing, but SmartScreen warnings are still a separate issue.
